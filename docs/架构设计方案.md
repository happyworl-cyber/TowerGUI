# 自研跨引擎 UI 库 — 架构设计方案

> **定位**：用 React(TSX) 写 UI，编译/运行到 Unity / Cocos / Godot 原生UI系统  
> FairyGUI 仅作为设计参考，**不是**运行时依赖  
> 分析时间：2026-03-12

---

## 一、整体思路

### 核心理念

```
开发者写 TSX  →  自研 UI 库  →  各引擎原生 UI 渲染
```

| 层次 | 职责 | 类比 |
|------|------|------|
| **TSX 源码层** | 开发者书写的声明式 UI 描述 | React JSX / SwiftUI |
| **核心抽象层** | 与引擎无关的 UI 节点树 + 布局 + 事件 | React VirtualDOM |
| **引擎适配层** | 将抽象节点映射到引擎原生对象 | react-native 的 Shadow Tree |
| **引擎原生层** | Unity UGUI / Cocos cc.Node / Godot Control | Platform Native UI |

### 从 FairyGUI 学到的核心设计思路（只借鉴思想，不用代码）

| FairyGUI 设计思想 | 我们的对应方案 |
|------------------|---------------|
| GObject 统一基类 | `UINode` 统一抽象节点 |
| Relations 布局关联 | **Flex 布局引擎**（更现代） |
| Controller + Gear 状态机 | **React 状态（useState/useReducer）** |
| Transition 动画系统 | **动画 API + CSS-like 过渡** |
| UIPackage 资产管理 | **资产注册表 + 异步加载** |
| GList 虚拟列表 | **VirtualList 组件（内置）** |

## 二、执行模式选择（关键决策）

### 模式 A：运行时执行（Runtime Mode）

```
TSX + React
    ↓ esbuild/Vite 打包
    JS Bundle
    ↓ Puerts (V8/QuickJS) 在引擎内运行
    自研 React Reconciler
    ↓ 引擎 Native UI API
    Unity UGUI / Cocos Node / Godot Control
```

**优点**：
- 完整 React Hooks、Context、Suspense 生态
- 支持**热更新**（不重启游戏更新 UI）
- npm 生态直接用（动画库、状态管理等）
- 开发调试用 Chrome DevTools

**缺点**：
- JS 运行时有内存和初始化开销（V8 约 5~10MB）
- 跨语言调用（JS↔C#）有性能损耗

**适合场景**：复杂交互UI（背包、商城、社交系统）、需要热更新的内容型界面

---

### 模式 B：编译时生成（AOT Compile Mode）

```
TSX 组件树
    ↓ 自研编译器（Babel/SWC 插件）
    静态分析 + 代码生成
    ↓
    ├── Unity：C# 代码 + Prefab 构建脚本
    ├── Cocos：TypeScript 组件代码
    └── Godot：GDScript / C# 场景文件
```

**优点**：
- **零运行时开销**，完全原生性能
- 生产包体积小
- 无需 JS 运行时依赖

**缺点**：
- 纯静态，动态 UI（条件渲染、循环列表变化）需要特殊处理
- 需要开发编译器工具链
- 调试链路更长

**适合场景**：固定布局的 HUD、简单状态面板、Loading 界面

---

### ✅ 推荐：两种模式共存，统一 API

```
同一套 TSX 代码
    ├── 开发模式 → Runtime Mode（热更新，快速迭代）
    └── 发布模式 → AOT Mode（可选，性能优化）
```

**第一阶段先做 Runtime Mode（最快落地，功能最完整），AOT 模式按需后续实现。**

使用 **Yoga**（Facebook 开源的 Flex 布局引擎，react-native 同款）的 WebAssembly 版本，或接入 `yoga-layout-prebuilt` npm 包：

```typescript
// packages/core/src/layout/FlexLayout.ts
import Yoga from 'yoga-layout-prebuilt';

export class LayoutNode {
  yogaNode = Yoga.Node.create();

  applyStyle(props: UINodeProps) {
    const { yogaNode } = this;
    if (props.width !== undefined)   yogaNode.setWidth(props.width as number);
    if (props.height !== undefined)  yogaNode.setHeight(props.height as number);
    if (props.flex !== undefined)    yogaNode.setFlex(props.flex);
    if (props.flexDirection)         yogaNode.setFlexDirection(FLEX_DIR_MAP[props.flexDirection]);
    if (props.padding !== undefined) applyPadding(yogaNode, props.padding);
    // ... 以此类推
  }

  calculate(width: number, height: number) {
    this.yogaNode.calculateLayout(width, height, Yoga.DIRECTION_LTR);
  }

  getComputedLayout() {
    return this.yogaNode.getComputedLayout();
    // 返回 { left, top, width, height }
  }
}
```

### 3.3 React Reconciler（核心桥接）

```typescript
// packages/core/src/reconciler.ts
import Reconciler from 'react-reconciler';
import { DefaultEventPriority } from 'react-reconciler/constants';
import type { IEngineAdapter, EngineNode } from './IEngineAdapter';

export function createReconciler(adapter: IEngineAdapter) {
  return Reconciler<
    string,          // Type
    UINodeProps,     // Props
    EngineNode,      // Container
    EngineNode,      // Instance
    EngineNode,      // TextInstance
    any, any, any, any, any, any, any, boolean, null
  >({

    // ===== 节点创建 =====
    createInstance(type, props) {
      const node = adapter.createElement(type);
      adapter.applyProps(node, {}, props);
      return node;
    },
    createTextInstance(text) {
      return adapter.createText(text);
    },

    // ===== 树操作 =====
    appendChildToContainer: adapter.appendChild.bind(adapter),
    appendChild:            adapter.appendChild.bind(adapter),
    insertBefore:           adapter.insertBefore.bind(adapter),
    removeChildFromContainer: adapter.removeChild.bind(adapter),
    removeChild:            adapter.removeChild.bind(adapter),

    // ===== 属性更新（差量）=====
    prepareUpdate(instance, type, oldProps, newProps) {
      return computeDiff(oldProps, newProps); // 只返回变化的 keys
    },
    commitUpdate(instance, updatePayload, type, oldProps, newProps) {
      adapter.applyProps(instance, oldProps, newProps, updatePayload);
    },
    commitTextUpdate(instance, oldText, newText) {
      adapter.updateText(instance, newText);
    },

    // ===== 标准配置 =====
    finalizeInitialChildren: ()  => false,
    getChildHostContext:     (ctx) => ctx,
    getRootHostContext:      ()  => ({}),
    getPublicInstance:       (i) => i,
    prepareForCommit:        ()  => null,
    resetAfterCommit:        ()  => {},
    shouldSetTextContent:    ()  => false,
    clearContainer:          (c) => adapter.clearChildren(c),
    supportsMutation:        true,
    isPrimaryRenderer:       false,
    getCurrentEventPriority: ()  => DefaultEventPriority,
    scheduleTimeout:  setTimeout,
    cancelTimeout:    clearTimeout,
    noTimeout:        -1,
    supportsMicrotasks: false,
    supportsPersistence: false,
    beforeActiveInstanceBlur: () => {},
    afterActiveInstanceBlur:  () => {},
    detachDeletedInstance:    () => {},
  });
}
```

## 四、各引擎适配器实现

### 4.1 Unity 适配器（Puerts + UGUI）

```typescript
// packages/unity-adapter/src/UnityAdapter.ts

const CS_UGUI = CS.UnityEngine.UI;
const CS_GO   = CS.UnityEngine.GameObject;
const CS_RT   = CS.UnityEngine.RectTransform;
const TMP     = CS.TMPro.TextMeshProUGUI;  // TextMeshPro

export class UnityAdapter implements IEngineAdapter {

  // 节点类型 → Unity GameObject 工厂
  private creators: Record<string, () => CS.UnityEngine.GameObject> = {
    'ui-view':    () => this.makeGO('View',     CS_UGUI.Image),
    'ui-scroll':  () => this.makeScroll(),
    'ui-text':    () => this.makeGO('Text',     TMP),
    'ui-input':   () => this.makeGO('Input',    CS_UGUI.InputField),
    'ui-image':   () => this.makeGO('Image',    CS_UGUI.RawImage),
    'ui-sprite':  () => this.makeGO('Sprite',   CS_UGUI.Image),
    'ui-button':  () => this.makeButton(),
    'ui-toggle':  () => this.makeGO('Toggle',   CS_UGUI.Toggle),
    'ui-slider':  () => this.makeGO('Slider',   CS_UGUI.Slider),
    'ui-progress':() => this.makeGO('Progress', CS_UGUI.Slider),
    'ui-list':    () => this.makeList(),
  };

  createElement(type: string) {
    const creator = this.creators[type];
    if (!creator) throw new Error(`Unknown UI type: ${type}`);
    return creator();
  }

  applyProps(go: CS.UnityEngine.GameObject, oldProps: any, newProps: any) {
    const rt = go.GetComponent(CS_RT) as CS.UnityEngine.RectTransform;

    // 布局
    if (newProps.width !== undefined || newProps.height !== undefined) {
      rt.sizeDelta = new CS.UnityEngine.Vector2(
        newProps.width  ?? rt.sizeDelta.x,
        newProps.height ?? rt.sizeDelta.y,
      );
    }

    // Flex 布局计算后应用坐标
    if (newProps._layoutLeft !== undefined) {
      rt.anchoredPosition = new CS.UnityEngine.Vector2(
        newProps._layoutLeft, -newProps._layoutTop  // UGUI Y轴向上
      );
    }

    // 文字
    if (newProps.text !== undefined) {
      const tmp = go.GetComponent(TMP);
      if (tmp) tmp.text = newProps.text;
    }

    // 透明度
    if (newProps.opacity !== undefined) {
      const cg = go.GetComponent(CS.UnityEngine.CanvasGroup)
               ?? go.AddComponent(CS.UnityEngine.CanvasGroup);
      cg.alpha = newProps.opacity;
    }

    // 可见性
    if (newProps.visible !== undefined) go.SetActive(newProps.visible);

    // 事件
    if (newProps.onClick && newProps.onClick !== oldProps?.onClick) {
      const btn = go.GetComponent(CS_UGUI.Button);
      if (btn) {
        btn.onClick.RemoveAllListeners();
        btn.onClick.AddListener(newProps.onClick);
      }
    }

    // 图片资源（异步加载）
    if (newProps.src && newProps.src !== oldProps?.src) {
      AssetLoader.loadSprite(newProps.src).then(sprite => {
        const img = go.GetComponent(CS_UGUI.Image);
        if (img) img.sprite = sprite;
      });
    }
  }

  // ... 其他方法
}
```

### 6.1 渲染优化层

FairyGUI 自建了渲染管线（FairyBatching），我们基于 UGUI，做**适配层级别**的优化：

```typescript
// packages/core/src/render/BatchOptimizer.ts

export interface BatchHint {
  /** 是否为此容器启用合批优化（对应 FairyGUI fairyBatching） */
  enableBatching?: boolean;
  /** 子节点渲染排序策略 */
  childrenRenderOrder?: 'ascent' | 'descent' | 'arch';
  /** arch 模式的顶点索引 */
  apexIndex?: number;
}

export interface IEngineAdapter {
  // ... 已有接口 ...

  /** 批量提交属性更新，减少 JS↔C# 跨边界调用次数 */
  commitBatch(updates: BatchUpdate[]): void;
  /** 设置合批标记 */
  setBatchingMode(node: EngineNode, enabled: boolean): void;
  /** 设置渲染排序 */
  setRenderOrder(node: EngineNode, order: number): void;
}

interface BatchUpdate {
  node: EngineNode;
  props: Record<string, unknown>;
}
```

**Unity 适配器实现要点**：
- 每帧收集所有 `commitUpdate` 调用，在帧末一次性通过 Puerts 批量提交给 C# 端
- C# 端接收 `BatchUpdate[]`，一次循环完成所有属性赋值，减少 JS↔C# 调用从 N 次降为 1 次
- 利用 UGUI Canvas 的合批机制，通过 `CanvasGroup` 和层级管理辅助 DrawCall 优化

```typescript
// packages/core/src/pool/EngineNodePool.ts

export class EngineNodePool {
  private pools: Map<string, EngineNode[]> = new Map();

  acquire(type: string, adapter: IEngineAdapter): EngineNode {
    const pool = this.pools.get(type);
    if (pool && pool.length > 0) {
      return pool.pop()!;
    }
    return adapter.createElement(type);
  }

  release(type: string, node: EngineNode, adapter: IEngineAdapter): void {
    adapter.resetNode(node);
    const pool = this.pools.get(type) ?? [];
    pool.push(node);
    this.pools.set(type, pool);
  }
}
```

## 十一、关键技术风险

| 风险 | 影响 | 对策 |
|------|------|------|
| Puerts JS↔C# 调用开销 | 高频属性更新（60帧动画）可能瓶颈 | 动画走 C# Unity Tween；批量提交 props 更新 |
| Yoga 布局计算开销 | 每帧大量节点重排 | 标脏机制，只重计算布局变化的子树 |
| IL2CPP AOT 限制 | iOS/主机平台 Puerts 需额外配置 | 用 Puerts 静态包装生成工具（已有） |
| React bundle 体积 | 初始加载耗时 | 用 Preact（3KB）替代 React（40KB）；代码分割 |
| GC 压力 | JS + C# 双端 GC，可能抖动 | 对象池复用 EngineNode；避免高频创建小对象 |
