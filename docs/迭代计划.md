**关键决策**：Preact 没有 `react-reconciler` 那样的独立包，需要用 `preact/options` 钩子拦截 VDOM 操作并转发到 engine adapter。或者使用 `preact` + 自定义渲染目标。

**AI 提示**：


## 风险应对预案

| 风险触发 | 应对措施 |
|---------|---------|
| Week 0 性能不达标 | 实现 C# 批量命令缓冲区重测；仍不行则放弃 Reconciler |
| Preact 自定义渲染器太难 | 回退到 React 18 + react-reconciler（成熟方案） |
| Yoga WASM 太慢 | 切 Yoga C++ Native Plugin 或纯 UGUI 布局 |
| UGUI Canvas Rebuild 严重 | 自动 Sub-Canvas 分割 + 考虑 UIToolkit 后端 |
| iOS 性能差 | QuickJS 后端 + 减少桥接调用 + 更多逻辑下沉 C# |
| Web Preview 与 Unity 效果差异大 | 提供 diff 标注 + 以 Unity 为准的说明 |
| Addressable 加载复杂度高 | 先用 Resources 验证，再渐进切 Addressable |
| 某个 Sprint 延期 | 允许 ±1 周弹性，不影响后续 Phase 启动 |
